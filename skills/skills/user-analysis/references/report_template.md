# 报告结构（V3）

## 版本说明

V3 版本：两章结构（数据总览 + 执行策略），P2 拆分羊毛党/非羊毛党，流失对比用保留三高数据，PDF 单页连续 + 自动目录。

## 文件命名

- 基础报告（骨架）：`reports/user_analysis_{YYYYMMDD}.md`
- 业务版（润色后）：`reports/action_analysis_{YYYYMMDD}.md`
- PDF 报告：`reports/action_analysis_{YYYYMMDD}.pdf`

## 业务版结构（润色后）

提示词定义在 `src/llm/prompts/action.py`，两章 + 附录结构：

```
一、数据总览
  数据总览结论（3条固定结论）
  1. 人群分布（含 P2 拆分行）
  2. 三高用户规模分析（新晋来源 + ARPU 归因 + 保留三高品类增量拆解 + 品类结构对比）
二、执行策略
  1. 三高用户明星单品推荐（按大类：生鲜/新场景/标品）
  2. 流失三高用户召回（对比保留三高数据）
  3. P1 用户升级策略
  4. P2 非羊毛党用户升级策略
  5. P3 用户召回策略
附录（转化提升度说明 + 升级率明细 + 核心品类明细）
```

### V3 关键规则

1. **P2 标签一致**：凡涉及 P2 品类/消费/升级，一律写"P2 非羊毛党"
2. **P2 拆行**：人群分布表中 P2 必须拆成非羊毛党和羊毛党两行
3. **流失对比口径**：流失三高品类占比对比必须用"保留三高"数据，禁止用"本期三高"全量
4. **加粗范围**：只加粗人群名称（如 **P3**），禁止将占比/品类纳入加粗
5. **第一章表格无说明**：表格下方不添加任何说明文字、注释、脚注
6. **策略人群标题**：升级根因/流失根因等标题用纯文本，不用 `**` 加粗

## PDF 生成

使用 `scripts/md_to_pdf.py`（底层调用 `scripts/md2pdf.mjs`），特性：
- 单页连续，不分页截断
- 自动生成 h2/h3 目录（带锚点跳转）
- 楷体样式，蓝色标题
