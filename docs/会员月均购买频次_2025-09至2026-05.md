# 会员月均购买频次

> 取数时间：2026-05-11
> 数据范围：2025-09 ~ 2026-05（数据库最早仅覆盖到 2025-09，无 2024 年数据）

## 取数逻辑

- **底表**：`default_catalog.ads_business_analysis.store_transaction_details`（订单商品明细，一行一个商品）
- **会员**：`thirdparty_user_identity` 去重计数（过滤空值）
- **订单**：`order_id` 去重计数
- **月均频次**：去重订单数 / 去重会员数
- **注意**：`order_id` 需去重，因为一个订单含多个商品会拆成多行

## SQL

```sql
SELECT
    DATE_FORMAT(DATE(pay_at), '%Y-%m') AS month,
    COUNT(DISTINCT order_id) AS order_count,
    COUNT(DISTINCT thirdparty_user_identity) AS member_count,
    ROUND(COUNT(DISTINCT order_id) * 1.0 / COUNT(DISTINCT thirdparty_user_identity), 2) AS avg_frequency
FROM default_catalog.ads_business_analysis.store_transaction_details
WHERE thirdparty_user_identity IS NOT NULL
  AND thirdparty_user_identity != ''
GROUP BY DATE_FORMAT(DATE(pay_at), '%Y-%m')
ORDER BY month
```

## 结果

| 月份 | 订单数 | 会员数 | 月均频次 |
|------|--------|--------|----------|
| 2025-09 | 7,836 | 3,641 | 2.15 |
| 2025-10 | 14,567 | 5,567 | 2.62 |
| 2025-11 | 16,646 | 6,317 | 2.64 |
| 2025-12 | 18,850 | 7,228 | 2.61 |
| 2026-01 | 18,848 | 6,506 | 2.90 |
| 2026-02 | 11,548 | 4,823 | 2.39 |
| 2026-03 | 19,771 | 6,706 | 2.95 |
| 2026-04 | 18,740 | 6,758 | 2.77 |
| 2026-05 | 7,288 | 3,922 | 1.86 |

> 2026-05 仅到 5 月 10 日，为不完整月。

## 简要观察

- 会员规模从 3,600 增长到 6,700+，接近翻倍
- 月均频次在 2.15 ~ 2.95 之间波动，整体约 2.6-2.7 次/月
- 2 月为低谷（春节假期），1 月和 3 月为高峰
- 该频次为全量会员简单平均，未排除仅购买一次的流失用户
